# Submitting TokenWatcher to the Microsoft Store

Once published, users install with one click from the Store — no SmartScreen dialogs, automatic updates, no code-signing cert needed (Microsoft signs the package).

## One-time setup

### 1. Create an Individual Microsoft Partner Center account (free)

1. Go to <https://partner.microsoft.com/dashboard/registration>
2. Sign in with your personal Microsoft account
3. Choose **Individual** account type (free since 2023; Company is $99/yr)
4. Verify identity (Microsoft sends a postal letter or instant verification depending on country)

### 2. Reserve the app name

1. Partner Center → **Apps and games** → **New product** → **MSIX or PWA app**
2. Enter `TokenWatcher` as the product name
3. Microsoft reserves the name and assigns three identity values you'll need:
   - **Package/Identity Name** (e.g., `12345Geha.TokenWatcher`)
   - **Publisher ID** (e.g., `CN=12345678-ABCD-...`)
   - **Publisher Display Name** (e.g., `Geha Hariharan`)

   Find these at: *Product → Product identity*.

### 3. Wire the identity into `package.json`

In `package.json`, replace the placeholders under `build.appx`:

```json
"appx": {
  "applicationId": "TokenWatcher",
  "displayName": "TokenWatcher",
  "identityName": "12345Geha.TokenWatcher",
  "publisher": "CN=12345678-ABCD-...",
  "publisherDisplayName": "Geha Hariharan",
  "backgroundColor": "transparent",
  "languages": ["en-US"]
}
```

### 4. Build the MSIX locally

```powershell
.\build-sidecar.bat       # ensure resources/claude_fetch.exe is up to date
npm run build:msix        # produces dist/TokenWatcher_0.1.0.0_x64.msixbundle
```

### 5. Upload to Partner Center

1. Partner Center → your TokenWatcher product → **Submission** → **Packages**
2. Drag & drop the `.msixbundle` from `dist/`
3. Fill in:
   - **Description** (use the README's "What it shows" + "How it works" sections)
   - **Screenshots** (at least one 1366×768 or larger PNG of the side panel)
   - **Store logo** (`build/appx/StoreLogo.png` already generated)
4. Set **Pricing & availability** → Free, choose markets (Worldwide is fine)
5. Submit for certification

Microsoft's automated cert tests typically take 30–60 minutes. Human review takes 1–3 days. Once certified, the app is live at `apps.microsoft.com/detail/<id>`.

## Notes on the AppX/MSIX build

- The Square150x150Logo, Square44x44Logo, etc. assets are auto-generated by `scripts/gen-icons.py` into `build/appx/` and consumed by electron-builder.
- electron-builder handles the AppX packaging via Microsoft's [makeappx.exe](https://learn.microsoft.com/en-us/windows/msix/package/manual-packaging-root) — no extra tools needed on Windows.
- The MSIX runs in a sandbox by default. Our use cases — reading `~/.codex/auth.json`, launching `msedge.exe`, calling DPAPI — all work in unrestricted mode (`runFullTrust` capability), which is granted to apps signed by trusted authorities (including the Microsoft Store).

## Updates

For each new version:
1. Bump `package.json` version
2. Tag and push (CI builds the NSIS installer)
3. Locally run `npm run build:msix` and upload the new `.msixbundle` to Partner Center → New submission
4. Microsoft handles delta updates and auto-rollout to existing users
